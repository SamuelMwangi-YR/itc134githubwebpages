Careers in software releases

By: Yuki Candelario

Positions:

Release managers- are responsible for planning what will be contained in a release, managing the software engineering process, and releasing the final product.

Build and release engineers- build and design software applications and manage version control. They also test and troubleshoot the code before release.

Release engineers- deal with the more technical features of turning a release from development to production.

General skills & qualifications:

Release managers- Candidates need a bachelor’s degree in computer science or related field, in between 2-6 years of previous experience, 8-10 years experience in information system operations, formal training and advanced knowledge of software development lifecycle.

Build and release engineers- Companies will be more inclined to hire candidates that hold a bachelor's and skills in communication, analytical thinking/problem solving, working with teams, project management, programming and developing.

Release engineers- Most employers prefer a bachelor’s and knowledge in programming, and troubleshooting. Some helpful skills include high attention to detail, communication, time and change management.
