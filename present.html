Present of Software Releases

By: Stefan Buruiana

Software release is the distribution of software to the consumer.

From floppy disks to CDs, one-off downloads to continuous delivery, software distribution speed has accelerated exponentially. While how software is “released” has dramatically changed, it still remains it’s distribution to the consumer.


Software releases have changed with the advancing of technologies and appearance of internet. First software development was managed in a Waterfall methodology, where whole process is divided in parts, and project is executed in stages and each stage must be completed in order to proceed with the next one. It works best when product requirements are well-defined up-front with a high certainty because once a stage is done it is very hard to go back and optimize it or even impossible.


With the appearance of internet, customers no longer had to wait a year or more for a software CD to be released – they could download it. Easier access to feedback gave developers the opportunity to speed up the production of patches, fixes and updates. Most bugs started to be spotted well before deployment by testing the code prior to that. The speed of software release took a huge leap forward creating Agile Methodology.


Cloud tools and services made provisioning infrastructure faster than ever. Developers can quickly try new things without waiting for IT operations to provision services to them. Cloud computing enabled teams to automate more and more of the process of software releasing, the connection between Development and Operations team began to break down due that, creating DevOps methodology.
